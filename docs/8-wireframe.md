# TodoListApp Wireframe

## 1. 문서 개요

- 기준 문서: `docs/1-domain-definition.md`, `docs/2-prd.md`, `docs/3-user-scenario.md`, `docs/99-uc.md`
- 목적: MVP 1차 릴리즈의 화면 구조와 주요 사용자 흐름을 ASCII 와이어프레임으로 정의한다.
- 대상 플랫폼: 반응형 웹 UI
- 주요 사용자: 20대부터 50대 직장인

## 2. 화면 목록

| ID | 화면 | 주요 목적 |
| --- | --- | --- |
| WF-001 | 로그인 | 기존 사용자의 인증 |
| WF-002 | 회원가입 | 신규 사용자 계정 생성 |

## 4. 화면별 와이어프레임

### WF-001 로그인

```text
+------------------------------------------------------+
| TodoListApp                                          |
+------------------------------------------------------+
|                                                      |

|              +----------------------------+          |
```

필수 요소:

- 로그인 성공 시 JWT를 Zustand 메모리 상태에 저장하고 할일 목록으로 이동

## 5. 주요 화면 흐름

### 5.1 신규 사용자 흐름

```text
[회원가입] -> [로그인] -> [할일 목록] -> [새 할일] -> [할일 목록]
```

### 5.2 기존 사용자 흐름

```text
[로그인] -> [할일 목록] -> [필터 적용] -> [완료 변경]
                              |
                              +-> [할일 수정]
                              |
                              +-> [할일 삭제]
```

### 5.3 카테고리 생성 흐름

```text
[할일 등록/수정] -> [카테고리 추가] -> [카테고리 생성] -> [할일 등록/수정]
```

### 5.4 내 정보 및 탈퇴 흐름

```text
[할일 목록] -> [내 정보] -> [정보 수정] -> [저장]
                      |
                      +-> [회원 탈퇴 확인] -> [탈퇴하기] -> [로그인]
```

## 6. 상태 및 오류 표현

| 상태 | 표현 방식 |
| --- | --- |
| 로딩 | 목록 영역 또는 버튼 내부에 `불러오는 중...` 표시 |
| 빈 목록 | `조건에 맞는 할일이 없습니다.`와 새 할일 버튼 표시 |
| 인증 만료 또는 JWT 없음 | 로그인 화면으로 이동 |
| 필수값 누락 | 해당 입력 필드 하단에 오류 메시지 표시 |
| API 실패 | 화면 상단 또는 폼 하단에 오류 메시지 표시 |
| 삭제 확인 | 삭제 전 확인 모달 또는 확인 영역 표시 |

## 7. 와이어프레임 검토 기준

- PRD의 MVP 화면 범위를 모두 포함한다.
- 할일 목록에서 카테고리, 종료예정일 기간, 완료 여부 필터를 모두 제공한다.
- 할일 등록 시 할일명과 카테고리를 필수로 표현한다.
- 기본 카테고리와 사용자 추가 카테고리의 구분이 가능하다.
- 내 정보 화면에서 이메일은 읽기 전용이고 이름 또는 닉네임만 수정 가능하다.
- 회원 탈퇴 시 데이터 즉시 삭제 안내와 확인 절차를 제공한다.
- JWT는 화면에 노출하지 않으며 Zustand 메모리 상태에만 저장한다.
- 데스크톱과 모바일에서 핵심 작업이 동일하게 가능하다.
