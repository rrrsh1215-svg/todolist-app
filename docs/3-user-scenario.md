# TodoListApp 사용자 시나리오

## 1. 개요

TodoListApp은 20대부터 50대까지의 직장인이 개인 계정으로 로그인하여 본인의 할일을 안전하게 관리하는 반응형 웹 기반 개인 할일 관리 서비스이다.

사용자는 데스크톱 브라우저와 모바일 브라우저에서 회원가입, 로그인, 로그아웃, 내 정보 관리, 회원 탈퇴, 할일 CRUD, 완료 여부 변경, 카테고리 기반 분류, 복합 필터링을 수행할 수 있다.

MVP 기본 카테고리는 `일반`, `업무`, `개인`, `학습`이다. 사용자는 본인만 사용할 수 있는 사용자 추가 카테고리를 생성할 수 있다.

## 2. 페르소나별 시나리오

### 2.1 일정이 많은 실무자: 김민준

- 연령대: 30대
- 직무: IT 서비스 기획자
- 사용 환경: 회사 PC와 모바일 브라우저
- 주요 니즈: 업무, 개인 일정, 학습 할일을 카테고리로 구분하고 마감일 기준으로 빠르게 확인한다.
- 불편 사항: 메신저, 메모 앱, 캘린더에 할일이 흩어져 완료 여부를 추적하기 어렵다.

#### SC-001 출근 직후 오늘 처리할 업무 확인

1. 김민준은 회사 PC에서 TodoListApp에 접속한다.
2. 이메일과 비밀번호로 로그인한다.
3. 시스템은 로그인 성공 시 JWT를 발급하고 할일 목록 화면을 표시한다.
4. 프론트엔드는 JWT를 Zustand store의 메모리 상태에 저장한다.
5. 김민준은 카테고리 필터에서 `업무`를 선택한다.
6. 종료예정일 기간을 오늘 날짜로 설정한다.
7. 완료 여부를 `미완료`로 설정한다.
8. 시스템은 김민준 본인의 업무 카테고리 중 오늘까지 처리해야 하는 미완료 할일만 표시한다.
9. 김민준은 필요한 할일의 설명과 종료예정일을 수정한다.
10. 처리 완료한 항목은 완료 상태로 변경한다.

#### SC-002 이동 중 모바일에서 완료 처리

1. 김민준은 퇴근길 모바일 브라우저로 TodoListApp에 접속한다.
2. Zustand store의 메모리 상태에 보관된 JWT가 유효하면 할일 목록이 표시된다.
3. 김민준은 `학습` 카테고리의 오늘 종료예정 할일을 확인한다.
4. 완료한 학습 항목의 상태를 `완료`로 변경한다.
5. 시스템은 변경된 완료 상태를 저장하고 목록에 반영한다.

### 2.2 개인 업무를 정리하려는 중간관리자: 박서연

- 연령대: 40대
- 직무: 영업팀 관리자
- 사용 환경: 데스크톱 브라우저 중심, 필요 시 모바일 브라우저
- 주요 니즈: 업무와 개인 할일을 분리하고, 기간별로 마감 예정 항목을 확인한다.
- 불편 사항: 협업 도구는 기능이 많아 개인 할일 관리에는 복잡하고 부담스럽다.

#### SC-003 주간 마감 업무 점검

1. 박서연은 월요일 오전 데스크톱 브라우저에서 로그인한다.
2. 할일 목록에서 종료예정일 기간을 이번 주 월요일부터 금요일까지로 설정한다.
3. 카테고리 필터에서 `업무`를 선택한다.
4. 완료 여부 필터를 `미완료`로 선택한다.
5. 시스템은 박서연 본인의 업무 할일 중 이번 주 마감 예정이며 완료되지 않은 항목만 표시한다.
6. 박서연은 필요한 항목의 할일명, 설명, 종료예정일을 수정한다.
7. 더 이상 필요 없는 할일은 삭제한다.

#### SC-004 사용자 카테고리 생성 후 할일 분류

1. 박서연은 기본 카테고리 외에 `팀관리` 카테고리가 필요하다고 판단한다.
2. 카테고리 생성 UI에서 `팀관리`를 입력하고 저장한다.
3. 시스템은 해당 카테고리를 박서연 계정에만 귀속하여 생성한다.
4. 박서연은 새 할일을 등록하면서 카테고리로 `팀관리`를 선택한다.
5. 이후 목록에서 `팀관리` 카테고리로 필터링하여 관련 할일만 확인한다.

### 2.3 모바일 사용 비중이 높은 20대 직장인

- 연령대: 20대
- 직무: 마케팅 담당자
- 사용 환경: 모바일 브라우저 중심
- 주요 니즈: 별도 앱 설치 없이 빠르게 가입하고 개인 할일을 등록한다.

#### SC-005 모바일에서 회원가입 후 첫 할일 등록

1. 사용자는 모바일 브라우저에서 TodoListApp에 접속한다.
2. 회원가입 화면에서 이메일, 비밀번호, 이름 또는 닉네임을 입력한다.
3. 시스템은 필수값과 이메일 중복 여부를 확인한 뒤 계정을 생성한다.
4. 사용자는 이메일과 비밀번호로 로그인한다.
5. 할일명, 설명, 종료예정일, 카테고리 `개인`을 입력한다.
6. 시스템은 할일을 현재 사용자 계정에 귀속하여 저장한다.
7. 할일 목록에는 현재 사용자 본인의 할일만 표시된다.

### 2.4 개인정보 관리에 민감한 50대 직장인

- 연령대: 50대
- 직무: 재무 담당자
- 사용 환경: 데스크톱 브라우저
- 주요 니즈: 계정 정보 확인, 이름 또는 닉네임 수정, 탈퇴 시 데이터 즉시 삭제

#### SC-006 내 정보 수정

1. 사용자는 로그인 후 내 정보 화면으로 이동한다.
2. 현재 등록된 이메일과 이름 또는 닉네임을 확인한다.
3. 이름 또는 닉네임을 수정한다.
4. 시스템은 요청 사용자가 본인인지 확인한다.
5. 수정 가능한 항목이면 변경 내용을 저장한다.
6. MVP에서 이메일과 비밀번호 변경은 제공하지 않는다.

#### SC-007 회원 탈퇴 및 데이터 삭제

1. 사용자는 내 정보 화면에서 회원 탈퇴를 선택한다.
2. 시스템은 탈퇴 시 계정, 할일, 사용자 추가 카테고리가 즉시 삭제됨을 안내한다.
3. 사용자가 탈퇴를 확정한다.
4. 시스템은 사용자 계정, 본인 소유 할일, 사용자 추가 카테고리를 즉시 삭제한다.
5. 사용자는 로그아웃 처리된다.
6. 탈퇴 후 기존 JWT로 보호 API에 접근하려는 요청은 실패한다.

## 3. 핵심 사용자 여정

### 3.1 회원가입 및 로그인

1. 사용자는 회원가입 화면에 접속한다.
2. 이메일, 비밀번호, 이름 또는 닉네임을 입력한다.
3. 시스템은 이메일 형식, 이메일 중복 여부, 필수값을 검증한다.
4. 계정 생성에 성공하면 사용자는 로그인한다.
5. 시스템은 인증 성공 시 JWT를 발급한다.
6. 프론트엔드는 JWT를 Zustand store의 메모리 상태에 저장하고 인증이 필요한 API 요청에 사용한다.
7. 사용자는 할일 목록, 내 정보, 카테고리 생성 등 인증 기능을 사용할 수 있다.

### 3.2 로그아웃

1. 사용자는 로그아웃을 선택한다.
2. 프론트엔드는 Zustand store의 메모리 상태에 저장된 JWT를 삭제한다.
3. 사용자는 비인증 상태가 된다.
4. 인증이 필요한 화면 또는 API에 접근하면 로그인 화면으로 안내된다.
5. MVP에서는 서버 측 JWT 블랙리스트 또는 강제 만료 처리는 제외한다.

### 3.3 할일 등록

1. 사용자는 로그인 상태에서 할일 등록 UI를 연다.
2. 할일명, 설명, 종료예정일, 카테고리를 입력한다.
3. 할일명과 카테고리는 필수로 입력한다.
4. 카테고리는 기본 카테고리 또는 본인이 생성한 사용자 추가 카테고리 중 선택한다.
5. 시스템은 할일을 현재 사용자 계정에 귀속하여 저장한다.
6. 저장된 할일은 목록 화면에 표시된다.
7. 초기 완료 여부는 `미완료` 상태로 관리된다.

### 3.4 할일 조회 및 복합 필터링

1. 사용자는 할일 목록 화면에 접속한다.
2. 시스템은 현재 로그인한 사용자의 할일만 조회한다.
3. 사용자는 카테고리 필터를 선택한다.
4. 사용자는 종료예정일 시작일과 종료일을 선택한다.
5. 사용자는 완료 여부를 `전체`, `완료`, `미완료` 중 선택한다.
6. 시스템은 선택된 조건을 조합하여 모든 조건을 만족하는 할일만 표시한다.
7. 필터 결과가 없으면 빈 상태 안내를 표시한다.

### 3.5 할일 수정, 삭제, 완료 여부 변경

1. 사용자는 본인의 할일 목록에서 특정 할일을 선택한다.
2. 시스템은 해당 할일이 현재 사용자 소유인지 확인한다.
3. 사용자는 할일명, 설명, 종료예정일, 카테고리, 완료 여부를 수정한다.
4. 시스템은 변경 내용을 저장하고 목록에 반영한다.
5. 사용자는 필요 없는 할일을 삭제할 수 있다.
6. 삭제된 할일은 목록과 필터 결과에서 제외된다.

### 3.6 사용자 카테고리 생성

1. 사용자는 카테고리 생성 UI를 연다.
2. 새 카테고리명을 입력한다.
3. 시스템은 카테고리를 현재 사용자 계정에 귀속하여 저장한다.
4. 생성된 카테고리는 해당 사용자에게만 표시된다.
5. 사용자는 이후 할일 등록 또는 수정 시 해당 카테고리를 선택할 수 있다.
6. 다른 사용자는 이 카테고리를 조회하거나 사용할 수 없다.

### 3.7 내 정보 조회 및 수정

1. 사용자는 내 정보 화면에 접속한다.
2. 시스템은 JWT를 검증하고 현재 사용자 정보를 조회한다.
3. 사용자는 이메일과 이름 또는 닉네임을 확인한다.
4. 사용자는 이름 또는 닉네임을 수정한다.
5. 시스템은 수정 권한과 입력값을 검증한 뒤 저장한다.
6. MVP에서는 이메일 변경과 비밀번호 변경은 제공하지 않는다.

### 3.8 회원 탈퇴

1. 사용자는 내 정보 화면에서 회원 탈퇴를 선택한다.
2. 시스템은 삭제 대상 데이터를 안내한다.
3. 사용자가 탈퇴를 확정하면 시스템은 계정, 할일, 사용자 추가 카테고리를 즉시 삭제한다.
4. 사용자는 로그아웃 처리된다.
5. 삭제 이후 기존 JWT로 인증 API를 호출하면 접근이 거부된다.

## 4. 예외 및 오류 시나리오

### 4.1 회원가입 오류

- 이메일 형식이 올바르지 않으면 계정 생성이 실패한다.
- 비밀번호가 비어 있거나 정책에 맞지 않으면 계정 생성이 실패한다.
- 이름 또는 닉네임이 비어 있으면 계정 생성이 실패한다.
- 이미 사용 중인 이메일로 가입하면 계정 생성이 실패한다.

### 4.2 로그인 및 인증 오류

- 이메일 또는 비밀번호가 올바르지 않으면 로그인에 실패한다.
- JWT가 없거나 유효하지 않은 경우 인증이 필요한 API 접근은 거부된다.
- 로그아웃 후 Zustand store의 JWT가 삭제되면 사용자는 인증 화면으로 이동한다.
- 탈퇴한 사용자의 기존 JWT로 API 접근을 시도하면 실패한다.

### 4.3 권한 및 데이터 분리 오류

- 사용자가 다른 사용자의 할일 ID로 조회, 수정, 삭제를 요청하면 실패한다.
- 사용자가 다른 사용자의 사용자 추가 카테고리에 접근하려 하면 실패한다.
- 할일 목록 조회 시 다른 사용자의 데이터가 포함되면 안 된다.
- 내 정보 조회 및 수정은 현재 로그인한 본인 정보에 대해서만 가능하다.

### 4.4 할일 등록 및 수정 오류

- 할일명이 비어 있으면 등록 또는 수정이 실패한다.
- 카테고리가 선택되지 않으면 등록 또는 수정이 실패한다.
- 존재하지 않는 카테고리를 선택하면 저장이 실패한다.
- 현재 사용자에게 속하지 않은 카테고리를 선택하면 저장이 실패한다.
- 종료예정일 형식이 유효하지 않으면 저장이 실패한다.

### 4.5 필터링 오류 및 빈 결과

- 카테고리, 종료예정일 기간, 완료 여부를 조합했을 때 결과가 없으면 빈 목록 상태를 표시한다.
- 종료예정일 시작일이 종료일보다 늦은 경우 유효하지 않은 기간으로 처리한다.
- 완료 여부 필터 값이 허용된 값이 아니면 요청이 실패한다.
- 필터링 결과에는 항상 현재 사용자 소유의 할일만 포함되어야 한다.

### 4.6 회원 탈퇴 오류

- 인증되지 않은 사용자는 회원 탈퇴를 요청할 수 없다.
- 탈퇴 처리 중 일부 데이터만 삭제되는 상태가 발생하지 않도록 계정, 할일, 사용자 추가 카테고리 삭제는 일관되게 처리되어야 한다.
- 탈퇴 완료 후 동일 계정의 개인 데이터는 즉시 조회 불가능해야 한다.

## 5. 수용 기준 연결

| 시나리오/여정 | 연결 수용 기준 | 확인 포인트 |
| --- | --- | --- |
| 회원가입 | AC-001 | 유효한 이메일, 비밀번호, 이름 또는 닉네임으로 계정 생성 가능. 중복 이메일 가입 실패 |
| 로그인 및 JWT 인증 | AC-002 | 로그인 성공 시 JWT 발급. JWT 없거나 유효하지 않은 보호 API 접근 차단 |
| 로그아웃 | AC-002 | 로그아웃 시 Zustand store의 메모리 JWT 삭제. 서버 측 블랙리스트는 MVP 제외 |
| 내 정보 조회/수정 | AC-003 | 본인 정보 조회 가능. 이름 또는 닉네임 수정 가능. 이메일/비밀번호 변경 제외 |
| 사용자 데이터 분리 | AC-004 | 본인의 할일과 사용자 카테고리만 조회 가능. 타 사용자 데이터 접근 실패 |
| 할일 등록 | AC-005 | 할일명과 카테고리 입력 시 등록 성공. 필수값 누락 시 실패 |
| 할일 수정/삭제 | AC-006 | 본인 소유 할일의 할일명, 설명, 종료예정일, 카테고리, 완료 여부 수정 가능. 삭제 가능 |
| 카테고리 생성 | AC-007 | 사용자 추가 카테고리 생성 가능. 생성한 사용자에게만 노출. 기본 카테고리 제공 |
| 할일 필터링 | AC-008 | 카테고리, 종료예정일 기간, 완료 여부 및 복합 조건으로 목록 조회 가능 |
| 회원 탈퇴 | AC-009 | 탈퇴 시 계정, 할일, 사용자 추가 카테고리 즉시 삭제. 기존 JWT 접근 실패 |
