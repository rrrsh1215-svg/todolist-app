# TodoListApp 사용자 시나리오

## 1. 개요

사용자는 데스크톱과 모바일 브라우저에서 회원가입, 로그인, 로그아웃, 내 정보 관리, 회원 탈퇴, 할일 CRUD, 상태 변경, 카테고리 기반 분류, 복합 필터링을 수행할 수 있다.

기본 카테고리는 `일반`, `업무`, `개인`, `학습`이며, 프론트엔드는 사용자 언어에 맞게 기본 카테고리를 표시한다. 사용자 추가 카테고리는 입력한 이름 그대로 표시한다.

## 2. 주요 시나리오

### SC-001 출근 직후 오늘 처리할 업무 확인

1. 사용자는 TodoListApp에 로그인한다.
2. 시스템은 JWT를 발급하고 할일 목록 화면을 표시한다.
3. 사용자는 카테고리 필터에서 `업무`를 선택한다.
4. 사용자는 종료예정일 기간을 오늘 날짜로 설정한다.
5. 사용자는 상태 필터에서 `등록` 또는 `진행중`을 선택한다.
6. 시스템은 조건에 맞는 본인 소유 할일만 표시한다.
7. 사용자는 필요한 할일을 편집 화면에서 수정한다.
8. 처리 완료한 항목은 편집 화면에서 상태를 `완료`로 변경한다.

### SC-002 모바일에서 할일 상태 변경

1. 사용자는 모바일 브라우저에서 로그인한다.
2. 할일 목록에서 수정할 항목의 `편집`을 선택한다.
3. 편집 화면에서 상태를 `진행중`, `완료`, `취소` 중 하나로 변경한다.
4. 시스템은 변경된 상태를 저장하고 목록에 반영한다.

### SC-003 사용자 카테고리 생성 후 할일 분류

1. 사용자는 기본 카테고리 외에 `팀관리` 카테고리가 필요하다고 판단한다.
2. 카테고리 생성 UI에서 `팀관리`를 입력하고 저장한다.
3. 시스템은 해당 카테고리를 현재 사용자 계정에만 귀속해 생성한다.
4. 사용자는 새 할일 등록 또는 수정 시 `팀관리`를 선택한다.
5. 목록에서 `팀관리` 카테고리로 필터링해 관련 할일만 확인한다.

### SC-004 내 정보에서 다크모드와 언어 설정

1. 사용자는 내 정보 화면에 접근한다.
2. 사용자는 다크모드 사용 여부를 변경한다.
3. 사용자는 표시 언어를 한국어, 영어, 일본어 중 하나로 선택한다.
4. 저장 후 시스템은 사용자 설정을 저장한다.
5. 다음 접속 시 저장된 다크모드와 표시 언어가 자동 적용된다.

### SC-005 회원 탈퇴

1. 사용자는 내 정보 화면에서 회원 탈퇴를 선택한다.
2. 시스템은 계정, 본인 소유 할일, 사용자 추가 카테고리가 즉시 삭제됨을 안내한다.
3. 사용자가 탈퇴를 확정하면 시스템은 관련 데이터를 삭제한다.
4. 기존 JWT로 보호 API 접근을 시도하면 실패한다.

## 3. 상세 사용자 여정

### 3.1 회원가입 및 로그인

1. 사용자는 회원가입 화면에서 이메일, 비밀번호, 이름 또는 닉네임을 입력한다.
2. 시스템은 필수값, 이메일 형식, 이메일 중복 여부를 검증한다.
3. 계정 생성 성공 후 사용자는 로그인한다.
4. 로그인 성공 시 JWT와 사용자 정보를 반환한다.
5. 프론트엔드는 JWT를 Zustand 메모리 상태에만 저장한다.

### 3.2 할일 등록

1. 사용자는 `새 할 일`을 선택한다.
2. 할일명, 설명, 종료예정일, 카테고리를 입력한다.
3. 할일명과 카테고리는 필수이다.
4. 시스템은 기본 상태 `registered`로 할일을 저장한다.
5. 저장 후 목록 화면으로 이동한다.

### 3.3 할일 조회 및 필터링

1. 시스템은 현재 로그인 사용자의 할일만 조회한다.
2. 사용자는 카테고리 필터를 선택할 수 있다.
3. 사용자는 종료예정일 시작일과 종료일을 선택할 수 있다.
4. 사용자는 상태 필터에서 전체, 등록, 진행중, 완료, 취소 중 하나를 선택할 수 있다.
5. 시스템은 모든 조건을 조합해 목록을 표시한다.

### 3.4 할일 수정 및 삭제

1. 사용자는 목록에서 `편집`을 선택한다.
2. 시스템은 해당 할일이 현재 사용자 소유인지 확인한다.
3. 사용자는 할일명, 설명, 종료예정일, 카테고리, 상태를 수정한다.
4. 상태 값은 `registered`, `in_progress`, `completed`, `canceled`만 허용한다.
5. 사용자는 필요 없는 할일을 삭제할 수 있다.

## 4. 예외 및 오류 시나리오

- 이메일 형식이 올바르지 않으면 회원가입 또는 로그인이 실패한다.
- 이미 사용 중인 이메일로 회원가입하면 실패한다.
- 잘못된 이메일 또는 비밀번호로 로그인하면 실패한다.
- JWT가 없거나 유효하지 않으면 보호 API 접근이 실패한다.
- 다른 사용자의 할일 또는 사용자 추가 카테고리에 접근하면 실패한다.
- 할일명 또는 카테고리가 없으면 할일 등록/수정이 실패한다.
- 존재하지 않거나 현재 사용자에게 허용되지 않은 카테고리를 선택하면 실패한다.
- 종료예정일 형식이 유효하지 않으면 실패한다.
- 상태 값이 허용된 값이 아니면 실패한다.

## 5. 수용 기준

| 항목 | 기준 |
| --- | --- |
| 회원가입 | 유효한 이메일, 비밀번호, 이름 또는 닉네임 입력 시 계정 생성 |
| 로그인 | 유효한 계정 정보 입력 시 JWT 발급 |
| 사용자 데이터 분리 | 본인의 할일과 사용자 추가 카테고리만 조회 가능 |
| 할일 등록 | 할일명과 카테고리 입력 시 등록 성공 |
| 할일 수정/삭제 | 본인 소유 할일의 정보와 상태 수정 가능, 삭제 가능 |
| 상태 관리 | 등록, 진행중, 완료, 취소 상태를 편집 화면에서 저장 가능 |
| 카테고리 생성 | 사용자 추가 카테고리는 생성 사용자에게만 노출 |
| 필터링 | 카테고리, 종료예정일 기간, 상태 조건으로 목록 조회 가능 |
| 개인 설정 | 다크모드와 표시 언어 저장 및 재접속 시 자동 적용 |
| 회원 탈퇴 | 계정, 할일, 사용자 추가 카테고리 즉시 삭제 |
