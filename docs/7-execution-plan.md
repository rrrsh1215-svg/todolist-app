# TodoListApp 실행계획

본 문서는 `docs/` 디렉토리의 도메인 정의서, PRD, 사용자 시나리오, 프로젝트 구조 원칙, 아키텍처 다이어그램, ERD와 `database/schema.sql`을 기준으로 MVP 구현 작업을 데이터베이스, 백엔드, 프론트엔드 단위로 분해한 실행계획이다.

## 1. 실행 원칙

- MVP 범위는 회원가입, 로그인, 로그아웃, 내 정보 조회/수정, 회원 탈퇴, 할일 CRUD, 완료 여부 변경, 카테고리 조회/생성, 복합 필터링으로 제한한다.
- 기술 스택은 React 19 + TypeScript + Zustand + TanStack Query, Node.js + Express REST API, PostgreSQL 17, `pg`, JWT로 고정한다.
- JWT는 프론트엔드 Zustand `authStore`의 메모리 상태에만 저장한다.
- JWT는 `localStorage`, `sessionStorage`, 쿠키에 저장하지 않는다.
- 핵심 의존성 흐름은 `DB 스키마 -> 백엔드 API -> 프론트엔드 연동 -> 통합 QA`이다.
- 프론트엔드 화면 골격과 공통 컴포넌트는 백엔드 API 구현과 병렬 진행할 수 있다.

## 8. 최종 수용 체크리스트

- [ ] 사용자는 회원가입할 수 있다.
- [ ] 사용자는 로그인할 수 있고 JWT를 발급받는다.
- [ ] JWT는 Zustand 메모리 상태에만 저장된다.
- [ ] JWT는 `localStorage`, `sessionStorage`, 쿠키에 저장되지 않는다.
- [ ] 로그아웃 시 JWT가 메모리에서 삭제된다.
- [ ] 인증 사용자는 내 정보를 조회하고 이름/닉네임을 수정할 수 있다.
- [ ] 인증 사용자는 회원 탈퇴할 수 있다.
- [ ] 회원 탈퇴 시 사용자 계정, 본인 소유 할일, 사용자 추가 카테고리가 즉시 삭제된다.
- [ ] 인증 사용자는 기본 카테고리를 조회할 수 있다.
- [ ] 인증 사용자는 사용자 추가 카테고리를 생성할 수 있다.
- [ ] 인증 사용자는 할일을 등록, 조회, 수정, 삭제할 수 있다.
- [ ] 인증 사용자는 할일 완료 여부를 변경할 수 있다.
- [ ] 할일 목록은 카테고리, 종료예정일 기간, 완료 여부로 필터링할 수 있다.
- [ ] 여러 필터 조건을 함께 적용할 수 있다.
- [ ] 사용자는 다른 사용자의 할일과 카테고리에 접근할 수 없다.
- [ ] 데스크톱과 모바일 브라우저에서 핵심 흐름이 사용 가능하다.
