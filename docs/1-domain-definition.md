# TodoListApp 도메인 정의서

## 1. 도메인 개요

TodoListApp은 인증된 개인 사용자가 자신의 할일을 저장하고, 카테고리와 종료예정일, 상태 기준으로 관리하는 개인용 할일 관리 애플리케이션이다.

핵심 도메인은 사용자별 데이터 분리, 할일 등록/조회/수정/삭제, 카테고리 기반 분류, 조건별 목록 조회이다.

## 2. 핵심 개념

### 사용자

서비스를 이용하는 개인 주체이다. 사용자는 본인 계정의 개인정보, 할일, 사용자 추가 카테고리만 관리할 수 있다.

### 할일

사용자가 수행해야 하는 작업 단위이다. 할일은 다음 정보를 가진다.

- 할일명
- 설명
- 종료예정일
- 카테고리
- 상태

할일 상태는 `등록`, `진행중`, `완료`, `취소` 4단계로 관리한다. API 값은 `registered`, `in_progress`, `completed`, `canceled`를 사용한다.

### 카테고리

할일을 분류하기 위한 기준이다. 카테고리는 기본 카테고리와 사용자 추가 카테고리로 구분한다.

- 기본 카테고리: `일반`, `업무`, `개인`, `학습`
- 사용자 추가 카테고리: 인증 사용자가 직접 생성한 개인 카테고리

기본 카테고리는 전역 데이터로 유지하고, 프론트엔드에서 사용자 언어에 맞게 표시한다. 사용자 추가 카테고리는 입력한 이름 그대로 표시한다.

### 사용자 설정

사용자는 내 정보 화면에서 다크모드 사용 여부와 표시 언어를 저장할 수 있다. 표시 언어는 `ko`, `en`, `ja`를 지원한다.

## 3. 업무 규칙

- 사용자는 회원가입과 로그인 후 인증 사용자로 전환된다.
- JWT는 프론트엔드 Zustand 메모리 상태에만 저장한다.
- 사용자는 본인의 할일, 개인정보, 사용자 추가 카테고리에만 접근할 수 있다.
- 회원 탈퇴 시 사용자 계정, 본인 소유 할일, 사용자 추가 카테고리는 즉시 삭제한다.
- 할일 등록 시 할일명과 카테고리는 필수이다.
- 할일은 하나의 카테고리에 속한다.
- 할일 목록은 카테고리, 종료예정일 기간, 상태 조건으로 필터링할 수 있다.
- 할일 목록에서는 상태를 직접 변경하지 않고, 할일 편집 화면에서 상태를 변경한다.

## 4. 주요 유스케이스

### 회원가입 및 로그인

사용자는 이메일, 비밀번호, 이름 또는 닉네임으로 계정을 만들고 로그인한다. 로그인 성공 시 백엔드는 JWT와 사용자 정보를 반환한다.

### 할일 등록

인증 사용자는 할일명, 설명, 종료예정일, 카테고리를 입력해 할일을 등록한다. 등록된 할일의 기본 상태는 `registered`이다.

### 할일 목록 조회

인증 사용자는 본인 소유 할일 목록을 조회한다. 시스템은 카테고리, 종료예정일 기간, 상태 필터를 적용해 조건에 맞는 할일만 반환한다.

### 할일 수정 및 삭제

인증 사용자는 본인 소유 할일의 할일명, 설명, 종료예정일, 카테고리, 상태를 수정하거나 삭제할 수 있다.

### 카테고리 추가

인증 사용자는 기본 카테고리 외에 본인만 사용할 사용자 추가 카테고리를 생성할 수 있다.

### 내 정보 수정

인증 사용자는 이름 또는 닉네임, 다크모드 사용 여부, 표시 언어를 수정할 수 있다. 이메일과 비밀번호 변경은 MVP 범위에서 제외한다.

## 5. 범위 제외

- OAuth Social 인증
- 조직 단위 공동 할일 관리
- 할일 공유 및 담당자 배정
- 반복 일정
- 알림 및 리마인더
- 파일 첨부
- 평가 및 활동 이력
- 외부 캘린더 연동
- 관리자 기능
