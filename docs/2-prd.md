## 2. 제품 개요

TodoListApp은 20대부터 50대 직장인이 본인 계정으로 로그인하여 개인 할일을 등록하고, 카테고리와 종료예정일, 완료 여부 기준으로 할일을 관리할 수 있는 반응형 웹 애플리케이션이다.

핵심 가치는 다음과 같다.

- 사용자별 할일 데이터 분리
- 인증 기반 개인 데이터 보호
- 카테고리 기반 할일 분류
- 종료예정일과 완료 여부를 활용한 할일 조회
- 데스크톱과 모바일 브라우저에서 사용할 수 있는 반응형 웹 UI

## 6. 주요 사용자 흐름

### 회원가입 및 로그인

1. 사용자는 이메일, 비밀번호, 이름 또는 닉네임을 입력해 회원가입한다.
2. 사용자는 이메일과 비밀번호로 로그인한다.
3. 시스템은 인증 성공 시 JWT를 발급한다.
4. 프론트엔드는 JWT를 사용해 인증이 필요한 API를 호출한다.
5. 프론트엔드는 발급받은 JWT를 Zustand store의 메모리 상태에만 저장한다.
6. 사용자가 로그아웃하면 프론트엔드는 Zustand store의 JWT를 삭제한다.

### 개인 정보 수정

1. 인증 사용자는 본인의 개인 정보를 조회한다.
2. 사용자는 이름 또는 닉네임을 변경한다.
3. 시스템은 요청자가 본인인지 확인한 뒤 정보를 저장한다.
4. MVP에서 이메일 변경과 비밀번호 변경은 제외한다.

### 할일 관리

1. 인증 사용자는 할일명, 설명, 종료예정일, 카테고리를 입력해 할일을 등록한다.
2. 사용자는 본인 할일 목록을 조회한다.
3. 사용자는 할일 정보를 수정하거나 삭제한다.
4. 사용자는 할일 완료 여부를 완료 또는 미완료로 변경한다.

### 카테고리 관리

1. 시스템은 모든 사용자에게 기본 카테고리인 일반, 업무, 개인, 학습을 제공한다.
2. 인증 사용자는 본인이 사용할 추가 카테고리를 생성한다.
3. 사용자는 기본 카테고리 또는 본인이 생성한 카테고리를 할일에 지정한다.

### 할일 필터링

1. 사용자는 할일 목록 화면에서 필터 조건을 선택한다.
2. 필터 조건은 카테고리, 종료예정일 기간, 완료 여부를 포함한다.
3. 시스템은 조건을 조합하여 본인 소유 할일만 반환한다.

## 10. 수용 기준

### AC-001 회원가입

- 사용자가 유효한 이메일, 비밀번호, 이름 또는 닉네임을 입력하면 계정이 생성되어야 한다.
- 이미 사용 중인 이메일로 회원가입하면 실패해야 한다.

### AC-002 로그인 및 인증

- 사용자가 올바른 이메일과 비밀번호를 입력하면 JWT가 발급되어야 한다.
- JWT가 없거나 유효하지 않은 요청은 인증이 필요한 API에 접근할 수 없어야 한다.
- 발급된 JWT는 Zustand store의 메모리 상태에 저장되어야 한다.
- 사용자가 로그아웃하면 Zustand store의 JWT가 삭제되어야 한다.
- MVP에서는 서버 측 JWT 블랙리스트 또는 강제 토큰 폐기를 구현하지 않는다.

### AC-003 개인 정보 수정

- 인증 사용자는 본인의 이름 또는 닉네임을 수정할 수 있어야 한다.
- MVP에서 이메일 변경과 비밀번호 변경은 지원하지 않아야 한다.

### AC-004 사용자 데이터 분리

- 인증 사용자는 본인의 할일과 사용자 추가 카테고리만 조회할 수 있어야 한다.
- 다른 사용자의 할일 ID로 조회, 수정, 삭제를 요청하면 실패해야 한다.

### AC-005 할일 등록

- 할일명과 카테고리를 입력하면 할일이 등록되어야 한다.
- 할일명이 비어 있거나 카테고리가 없으면 등록이 실패해야 한다.

### AC-006 할일 수정 및 삭제

- 인증 사용자는 본인 소유 할일의 할일명, 설명, 종료예정일, 카테고리, 완료 여부를 수정할 수 있어야 한다.
- 인증 사용자는 본인 소유 할일을 삭제할 수 있어야 한다.

### AC-007 카테고리 생성

- 인증 사용자는 본인 계정에 귀속되는 사용자 추가 카테고리를 생성할 수 있어야 한다.
- 생성된 사용자 추가 카테고리는 다른 사용자에게 노출되지 않아야 한다.
- 기본 카테고리인 일반, 업무, 개인, 학습은 모든 인증 사용자에게 조회되어야 한다.

### AC-008 할일 목록 필터링

- 카테고리 필터를 적용하면 해당 카테고리의 할일만 조회되어야 한다.
- 종료예정일 기간 필터를 적용하면 해당 기간에 포함되는 할일만 조회되어야 한다.
- 완료 여부 필터를 적용하면 완료 또는 미완료 상태에 맞는 할일만 조회되어야 한다.
- 여러 필터를 함께 적용하면 모든 조건을 만족하는 할일만 조회되어야 한다.

### AC-009 회원 탈퇴

- 인증 사용자가 회원 탈퇴를 요청하면 사용자 계정과 관련 할일, 사용자 추가 카테고리가 즉시 삭제되어야 한다.
- 탈퇴한 사용자는 기존 JWT로 보호된 API에 접근할 수 없어야 한다.

## 11. API 범위

1차 릴리즈에서 필요한 REST API 범위는 다음과 같다.

- Auth API: 회원가입, 로그인, 로그아웃
- User API: 내 정보 조회, 내 정보 수정, 회원 탈퇴
- Todo API: 할일 등록, 목록 조회, 상세 조회, 수정, 삭제, 완료 여부 변경
- Category API: 기본 카테고리 및 사용자 추가 카테고리 조회, 사용자 추가 카테고리 생성

## 15. 주요 가정

- 1차 릴리즈의 인증 방식은 JWT만 사용한다.
- JWT는 Zustand store의 메모리 상태에 저장한다.
- 로그아웃은 Zustand store의 JWT 삭제 방식으로 처리하며, 서버 측 JWT 블랙리스트는 MVP 범위에서 제외한다.
- JWT는 `localStorage` 또는 `sessionStorage`에 저장하지 않는다.
- 종료예정일은 날짜 단위로 관리한다.
- 할일 상태는 완료와 미완료만 제공한다.
- 기본 카테고리는 일반, 업무, 개인, 학습이며 전역 카테고리로 관리한다.
- 사용자 추가 카테고리는 생성한 사용자에게만 귀속된다.
- MVP에서 개인 정보 수정 범위는 이름 또는 닉네임으로 제한한다.
- 3일 일정상 카테고리 수정/삭제는 1차 릴리즈 필수 범위에서 제외한다.
